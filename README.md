# 钢铁前线 · 指挥部

**单机 PvE** 红警风格网页经营游戏：采集资源 → 升级建筑 → 训练 / 进阶军队 → 出征打野怪。
不联网、不和玩家对战。原生 JS + ES Modules，零依赖、零构建，美术全是手写 SVG。

---

## 核心玩法循环

1. **采集资源**（中等节奏）：矿石场 / 石油场 / 冶炼厂 / 基地 / 科研实验室 产出
   矿石 · 石油 · 金属 · 电力 · 科研。**仓库**只存不产，提供巨量存储上限。
   顶部资源条以「数字 + 进度条」显示当前量 / 上限。
2. **升级建筑**（耗时 + 进度条）：每次升级消耗全部 5 种资源，成本随等级指数上升（统一 ×1.18/级）。
3. **训练军队**（耗时 + 进度条）：兵营 / 战车工厂 / 飞机场各 10 档兵种，每升 10 级解锁更强一档；
   建筑升级会加快训练。
4. **兵种进阶**（科研 + 耗时）：每个兵种可 1→10 阶，每阶 +12% 攻击与生命；
   进阶消耗科研、**耗时为该兵种训练时间的 20 倍**。各兵种阶级独立——新解锁的兵种从 1 阶起，
   与已进阶的旧兵种互不影响。
5. **出征打野怪**：100 关（每 10 关一页），胜利得战利品并解锁下一关，战斗损耗兵力。

界面三页：**基地** / **军队**（分建筑分组，含进阶）/ **征战**（100 关分页）。
右下角有可折叠的快捷导航（⚡ → 建筑/兵种/野怪），一键切页无需滚到顶部。进度条为能量流动样式。

## 数值设计（都在 `config.js`，可调）

- **建筑成本**：统一 `costGrowth = 1.18`，基地的单资源成本最高(200)。
  **各建筑主消耗的资源各不相同**(基地→电、仓库/兵营→矿、矿石场/战车厂→金、石油场/飞机场→油、冶炼厂/实验室→研)，
  使五种资源的总消耗大致均等(最高/最低仅差约 1.3 倍)，不再出现某种资源被抽干、其它堆满的情况。
- **仓库容量曲线**：`cap(1)=10000`，第 k→k+1 级增量 `1000 × 1.1393^(k-1)`。
  这样**仓库满级容量 ≈ 基地满级单次升级花费 / 0.9**，其余建筑的满级花费落在仓库满级容量的
  **72%~90%**（用脚本对齐过）。早期增量约 +1000 / +1139 / +1298 / +1479…（几何增长）。
- **兵种解锁**：每建筑 10 档，解锁等级 1, 10, 20 … 90。
- **训练耗时**：随档位上升；建筑每级约加快 5%。
- **进阶**：1~10 阶，成本 = 科研（随阶数与档位上升）；**耗时随阶数递增** = 训练耗时 × 20 × 1.6^(阶-1)。
- **初始资源**：各 10000，避免开局因建造顺序卡死。**基地升级最耗电力**（power 成本全场最高）。
- **野怪难度（中高）**：早期小队基础兵可过，中期需规模或进阶，后期必须高档/高阶兵——基础兵海也平推不了。

> 在建 / 在训 / 在进阶 都按绝对时间戳存档，离线到点会在下次打开时自动补完。

## 运行

```bash
cd resource-builder-game
python -m http.server 8000      # Windows: py -m http.server 8000
```
浏览器开 `http://localhost:8000`；Cloudflare Pages 直接 Direct Upload 整个文件夹。
